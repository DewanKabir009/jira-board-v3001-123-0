PW-01

Playwright runner contract

Completed implementation contract for the v3001.123.0 dashboard automation pilot.

Goal

The dashboard can request approved Playwright jobs, but it cannot run browser automation directly. A protected hosted runner owns execution, logs, screenshots, videos, traces, and final evidence.

Contract

Input

Job request

Must match job-contract.schema.json. The request includes script id, ticket key, environment, user, parameters, and artifact plan.

Control

Script registry

Only scripts listed in script-registry.json can run. The dashboard never accepts arbitrary code.

Output

Job record

The runner returns job id, status, current step, artifact links, failure reason, and timestamps.

Required endpoints

POST /playwright/jobs
GET  /playwright/jobs/{jobId}
GET  /playwright/jobs/{jobId}/events
POST /playwright/jobs/{jobId}/cancel

Acceptance

  • The 123 board links to this contract above the Data Pull section.
  • A job request cannot be created without an approved script id and ticket key.
  • Every job produces at least logs and a summary JSON artifact.
  • Video and trace are enabled for browser runs unless the script registry marks them unavailable.
  • Failed jobs keep enough evidence to debug without rerunning blind.
