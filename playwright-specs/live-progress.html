PW-04

Observable job progress

The dashboard polls the job summary after the runner publishes it and links directly to the Actions run while execution is active.

Implementation

  • Bridge response includes statusUrl, jobUrl, and actionsUrl.
  • The dashboard polls summary JSON and updates status, current step, artifacts, and failure reason.
  • Runner writes events.ndjson and logs.txt for deeper review.

Acceptance

A user can tell whether a run is queued, running, complete, or failed, and can open the exact evidence without terminal access.
