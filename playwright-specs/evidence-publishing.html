PW-05

Evidence publishing

Each job writes a durable evidence folder that GitHub Pages can serve to the dashboard and to Jira reviewers.

Published artifacts

  • summary.json for dashboard polling.
  • index.html for human review.
  • logs.txt and events.ndjson for traceability.
  • screenshot.png, trace.zip, and video.webm when the browser run produces them.

Acceptance

The final job record keeps enough evidence to debug or attach to Jira without rerunning the automation blindly.
